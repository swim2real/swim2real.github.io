Swim2Real: VLM-Guided System Identification for Sim-to-Real Transfer

Under Review

TL;DR: Swim2Real turns VLM diagnoses into validated simulator updates, reaching 55 mm in all five seeds with less than half the VLM rounds of Vid2Sid.

Abstract

We present Swim2Real, a pipeline for calibrating a robotic fish simulator from swimming videos. The method asks a vision-language model (VLM) to compare simulated and real motion, propose physically motivated updates to 16 simulator parameters, and pass each proposal through an objective-gated backtracking line search. Across five 40-evaluation seeds, Swim2Real reaches 51.3 +/- 1.2 mm final marker error, comparable to Bayesian Optimization at 52.4 +/- 2.1 mm, and reaches a 55 mm threshold in all seeds by evaluation 35. The line search reduces VLM usage relative to Vid2Sid (15.6 +/- 0.9 vs. 39.0 rounds per seed) while raising proposal acceptance from 14% to 42%. Velocity, simulated-RL, and open-loop replay checks support the calibrated simulator beyond the marker-error objective.

Method Detail

VLM Interface and Objective Gate

Swim2Real treats the VLM output as a candidate direction, not as an accepted calibration. Each VLM round receives side-by-side simulated and real marker overlays, the real video at the worst-matching frequency, per-marker and per-frequency errors, velocities, the current 16-parameter vector, fixed bounds, and recent proposal outcomes.

The response is parsed into a physical diagnosis, parameter names, proposed values, and a confidence estimate. The parser accepts only known parameter names, clamps proposed values to fixed bounds, and leaves unmentioned parameters unchanged. Backtracking then checks the full proposal and two smaller steps under the same 40-evaluation simulation budget. Only objective-improving candidates are accepted, so the VLM supplies physical hypotheses while the marker objective selects the calibration step.

Calibration

Reliability and VLM Efficiency

The main calibration comparison uses the same five initial seeds and a 40-evaluation simulation budget for Swim2Real, Bayesian Optimization, and Vid2Sid. Final marker errors are close, so the comparison emphasizes threshold reliability and VLM usage rather than final-error superiority.

Dynamics Check

Forward Swimming

Forward swimming is outside the marker-error objective. This comparison checks whether the calibrated simulator preserves swimming behavior beyond the calibration metric.

Out-of-Objective Checks

Velocity and Simulated RL

These checks test behavior not used by the calibration objective: velocity matching and simulated forward-swimming policy performance.

Hardware Check

Open-Loop Hardware Replay

The hardware check uses open-loop motor-command execution. The physical fish and MuJoCo simulator replay the same stored command sequence.
